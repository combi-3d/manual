    Segmentation by ilastik

    CoMBIで撮影したブロック面、特定の構造物（領域）をSegmentationしたい。

    1. Create New Projectより、Pixel Classification。プロジェクトファイル、拡張子ilpとして保存する。（右のカラムの1番から5番に従って進めます）
    2. 「1. Input Data」
      • 学習させるための画像を開く。CoMBIの連続画像のうち、全体を代表できるような数枚を選ぶ。Drag and Drop または、プルダウンメニューAdd New> Add separate image(s)
    3. 「2. Feature selection」　
      • 全部にチェックするので良いが、マシンパワー（CPU）によっては、減らした方がよい。カラム下方にリストが出てきて、各種フィルタがかかった状態を確認できる。
    4. 「3. Training」
      • Label1, Label2の二種類に塗り分けてみる。（多種類できるがマシンパワーが必要）。
      • 選び出したい領域を、Label1の黄色いペンで印をいれる。
      • それ以外の領域をLabel2青いペンで印を入れる。完全に塗りつぶす必要はないが、丁寧にやったほうが結果が良い。
      • Live Updateで結果を様子見しながら、作業できる。
      • 要不要？（Live OFFにして、Suggest Features, Filter Method (recommended), Number of Feartures 0 (auto), Run Feature Selection。しばらく待つ。下段のSelect Feature Set。）
    5. 「4. Prediction Export」
      • Source: とりあえずSimple Segmentation　白黒ハッキリしたやつ、二値化（2階調, 1 bit）情報になる。
      • ボタンChoose Export Image Settingsより、
        • yxcはチェックのまま、らしい。
        • Transformationでは、チェックなし、らしい。
        • Format: tif がよいらしい。白黒ハッキリしたいので。たぶんJPEGでは境界がへんになりそう。
        • File: では、保存先を指定する。よくわからないので、デフォルトの{dataset_dir}/{nickname}_{result_type}.tifを入力する。これで出力Exportされるファイルは、「同じフォルダに保存され、同じファイル名_Simple Segmentation.tif」になる。
      • 学習に使用した数枚の画像だけで、セグメンテーション画像が欲しい場合：　上カラムのExportボタンまたは左のExport Allボタンをおして、作業は終了する。
      • CoMBIの連続画像に同一処理のセグメンテーション作業を施す場合：　次の項目「5」へ。

    6. 「5. Batch Processing」

      • Select RAW Data Filesより、連続画像を選ぶ。Shiftを押しながら、複数画像を選択する。
      • 左カラムのボタン、Process all files。しばらく待つ。結構掛かるし、フリーズもした。マシンパワーに自信がなければ、少しずつ（100枚ずつなど）試してみる。
        • 出力されるtifファイルは8bit(0-255)であるが、二値化された情報を、0と1に入れている。つまり、0は黒、1は白。8bitで表記すると、00000000と00000001になる。それぞれ、学習時のデフォルトLabelのうち、Label1-黄色と、Label2-青色である。
        • Finderで表示されるアイコンは真っ黒画像になるのでビックリするが、アイコンはそれでよい。
        • 「4」での設定した条件のうち、Export Image SettingsのTransformation, renormalizeは、0,1を0,255に変換する機能のようだが、働かなかった。設定した条件はプロジェクトファイル（.ilp）には保存されないので、「5」を、別の日、別のパソコンで実行する場合は、注意すること（Mac OS 11, ilastik 1.4.0b, 正式対応していないバージョン）。
        • のちに別アプリで「扱いやすい二値化画像」にする（0,1ではなく、0, 255にする。下記） 。
        • （考え中；0,1をそのまま3D slicerに持って行っていいのか？？？）

    7. ファイルを整理する

      • MacならFinderで、リスト表示as List、
      • ファイルの種類で並べ替えKind、元画像JPEGと、二値化した画像tifで分かれる。
      • tifだけ選んで、新しいフォルダに収める。

    8. ImageJ/FIJIやPhotoshopで、利用しやすい二値化画像にする。

      • ImageJ/FIJIの場合
        • Image> Adjust> Threshold...
        • Dark Backgroundにチェックして、数値は2, 255。選択部位が白、背景が黒。（もし、チェックなしで、数値を、0, 1とすると、選択部位が黒、背景白になり、反転した画像になる。）
        • Apply, Save as
        • （Macroを使って、連続画像を処理する）
      • Photoshopの場合
        • Image> Adjustments> Threshold
        • 指定する数値は、2
        • （ActionとBatchを使って連続画像を処理する。）

    9. 3D slicerに入れて、Segmentation、3D再構築。もしくは、SegmentationしないならVolume Renderingだけでも。
